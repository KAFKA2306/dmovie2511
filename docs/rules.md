## 基本
- アセットは登録レジストリから取得し手動配置しない。
- エラーハンドリングコードは追加しない。
- エントリポイントはリポジトリルートに置き、補助モジュールは `ComfyUI/` 配下へ格納する。
- カスタムノードは `ComfyUI/custom_nodes/` 以下へ配置する。
- キュー管理の下ですべてのジョブを実行する。
- コメントは追加しない。
- テストとリンティングは CI とローカルの両方で実施する。
- テンプレートやプリセットはタグ付きで保存し再利用を優先する。
- ハードコーディングせず設定ファイルまたはメタデータから取得する。
- フォルダ構造に従いモデルは `ComfyUI/models/<category>/` へ収納する。
- ブリーフから成果物登録までの処理を全自動で完了させる。
- メタデータストアへ入出力パラメータを必ず記録する。
- ルート直下にはエントリポイント以外のファイルを置かない。
- ワークフロー定義は JSON で一元管理する。
- 一時出力は `ComfyUI/input/` と `ComfyUI/output/` を使用し、ルートに残さない。
- 依存関係は `uv sync` で管理し、ローカル変更を避ける。
- 追加コードは最小限に抑える。
- 全生成対象は健康的な成人とする。
- 新設ディレクトリは小文字スネークケースで命名し、docs に用途を記録する。
- 新設ディレクトリへ不要ファイルを置かない。
- モデル・LoRA は登録済みバージョンのみ利用する。
- automation/ は自動化ロジックと補助モジュールを保持する。
